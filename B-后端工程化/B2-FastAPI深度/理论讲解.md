## 1. FastAPI 核心架构

### 1.1 ASGI 与 Uvicorn

FastAPI 构建在 **ASGI**（Asynchronous Server Gateway Interface）之上，这是 WSGI 的异步继任者。

```
客户端请求 → Uvicorn(ASGI Server) → FastAPI(ASGI Application) → 路由匹配 → 依赖注入 → 端点函数 → 响应
```

- **Uvicorn**：高性能 ASGI 服务器，基于 uvloop + httptools，负责网络 I/O
- **Starlette**：FastAPI 的底层框架，提供路由、中间件、WebSocket 等基础设施
- **FastAPI**：在 Starlette 之上增加了类型标注驱动的参数解析、依赖注入、自动文档生成

### 1.2 路由系统

FastAPI 的路由系统支持 **装饰器注册** 和 **APIRouter 模块化**：

```python
from fastapi import FastAPI, APIRouter

app = FastAPI()

# 直接注册
@app.get("/items/{item_id}")
async def read_item(item_id: int):
    return {"item_id": item_id}

# APIRouter 模块化（推荐）
router = APIRouter(prefix="/api/v1", tags=["users"])

@router.get("/users")
async def list_users():
    return []

app.include_router(router)
```

路由匹配规则：
- 按注册顺序匹配，**先注册先匹配**
- 固定路径优先于路径参数（`/users/me` 优先于 `/users/{user_id}`）
- 支持路径参数类型约束：`{item_id:int}`、`{file_path:path}`

### 1.3 请求生命周期

一个请求从进入到返回的完整流程：

```
1. Uvicorn 接收 TCP 连接
2. 解析 HTTP 请求，构造 ASGI scope
3. 进入中间件链（洋葱模型，从外到内）
4. 路由匹配
5. 解析依赖（依赖注入树，深度优先）
6. 参数校验（Pydantic 验证路径参数、查询参数、请求体）
7. 执行端点函数
8. 序列化响应（Pydantic 模型 → JSON）
9. 中间件链返回（从内到外）
10. Uvicorn 发送 HTTP 响应
```

关键点：
- **中间件**是洋葱模型，请求时从外到内，响应时从内到外
- **依赖注入**在路由匹配后、端点执行前完成
- 校验失败会直接返回 **422 Validation Error**，不会进入端点函数

## 2. 依赖注入（Dependency Injection）

依赖注入是 FastAPI 最强大的特性之一，用于代码复用、关注点分离。

### 2.1 Depends() 基础

```python
from fastapi import Depends

def common_params(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

@app.get("/items")
async def read_items(params: dict = Depends(common_params)):
    return params
```

依赖可以是：
- **普通函数**（def）
- **异步函数**（async def）
- **可调用类**（实现 `__call__` 的类）

### 2.2 嵌套依赖

依赖可以递归依赖其他依赖，FastAPI 自动解析依赖树：

```python
def get_db():
    db = SessionLocal()
    try:
        yield db          # yield 之前是"注入"，之后是"清理"
    finally:
        db.close()

def get_current_user(db: Session = Depends(get_db)):
    # 依赖 get_db
    user = db.query(User).first()
    return user

def get_admin_user(user: User = Depends(get_current_user)):
    # 依赖 get_current_user，间接依赖 get_db
    if not user.is_admin:
        raise HTTPException(403, "Not admin")
    return user

@app.get("/admin/dashboard")
async def admin_dashboard(admin: User = Depends(get_admin_user)):
    return {"msg": f"Welcome {admin.name}"}
```

### 2.3 数据库 Session 注入

`yield` 依赖是管理资源生命周期的标准模式：

```python
from contextlib import contextmanager

def get_db():
    """每个请求获取独立的数据库 session，请求结束自动关闭"""
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

- `yield` 之前的代码在请求处理前执行（获取资源）
- `yield` 之后的代码在响应发送后执行（释放资源）
- 即使端点抛出异常，`finally` 也会执行

### 2.4 认证依赖

```python
from fastapi.security import OAuth2PasswordBearer

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="token")

async def get_current_user(token: str = Depends(oauth2_scheme)):
    """从 Authorization: Bearer <token> 中提取并验证 token"""
    user = decode_token(token)
    if not user:
        raise HTTPException(401, "Invalid token")
    return user
```

依赖注入的核心优势：
- **可测试性**：测试时可以用 `app.dependency_overrides` 替换依赖
- **复用性**：同一个依赖可被多个端点共享
- **组合性**：依赖可以自由嵌套组合

## 3. 中间件

中间件在请求到达路由之前和响应返回之前执行，采用洋葱模型。

### 3.1 CORS 中间件

```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],  # 允许的前端域名
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 3.2 自定义中间件

两种写法：

**装饰器写法**（简单场景）：
```python
@app.middleware("http")
async def log_requests(request: Request, call_next):
    start = time.time()
    response = await call_next(request)
    duration = time.time() - start
    print(f"{request.method} {request.url.path} - {duration:.3f}s")
    return response
```

**类写法**（复杂场景，推荐）：
```python
from starlette.middleware.base import BaseHTTPMiddleware

class RateLimitMiddleware(BaseHTTPMiddleware):
    def __init__(self, app, max_requests: int = 100):
        super().__init__(app)
        self.max_requests = max_requests
        self.requests = {}

    async def dispatch(self, request: Request, call_next):
        client_ip = request.client.host
        # 限流逻辑...
        response = await call_next(request)
        return response
```

### 3.3 中间件执行顺序

```python
app.add_middleware(A)  # 最后添加 = 最先执行
app.add_middleware(B)
app.add_middleware(C)  # 最先添加 = 最后执行

# 请求顺序：A → B → C → 路由
# 响应顺序：C → B → A → 客户端
```

注意：`add_middleware` 的添加顺序和执行顺序是**相反**的。最后添加的中间件最先接触到请求。

## 4. Pydantic V2 深度

Pydantic V2 是 FastAPI 的数据校验核心，用 Rust 重写了校验层，性能提升 5-50 倍。

### 4.1 模型定义与字段校验

```python
from pydantic import BaseModel, Field

class User(BaseModel):
    name: str = Field(..., min_length=2, max_length=50, description="用户名")
    age: int = Field(ge=0, le=150, description="年龄")
    email: str = Field(pattern=r'^[\w.-]+@[\w.-]+\.\w+$')
    tags: list[str] = Field(default_factory=list)
```

`Field()` 参数：
- `...`（Ellipsis）：必填字段
- `default` / `default_factory`：默认值
- `min_length` / `max_length`：字符串长度
- `ge` / `gt` / `le` / `lt`：数值范围
- `pattern`：正则校验
- `description`：写入 OpenAPI schema

### 4.2 嵌套模型

```python
class Address(BaseModel):
    city: str
    street: str

class Company(BaseModel):
    name: str
    address: Address  # 嵌套

class UserProfile(BaseModel):
    user: User
    company: Company | None = None  # 可选嵌套
    scores: list[float]             # 列表
```

### 4.3 自定义验证器

Pydantic V2 使用 `field_validator` 和 `model_validator`：

```python
from pydantic import field_validator, model_validator

class Order(BaseModel):
    price: float
    quantity: int
    discount: float = 0.0

    @field_validator("price")
    @classmethod
    def price_must_be_positive(cls, v):
        if v <= 0:
            raise ValueError("价格必须大于0")
        return v

    @model_validator(mode="after")
    def check_discount(self):
        if self.discount > self.price * self.quantity:
            raise ValueError("折扣不能超过总价")
        return self
```

- `field_validator`：校验单个字段，`mode="before"` 在类型转换前执行，默认 `mode="after"`
- `model_validator`：校验整个模型，可以访问多个字段做交叉校验

### 4.4 模型继承

```python
class UserBase(BaseModel):
    name: str
    email: str

class UserCreate(UserBase):
    password: str          # 创建时需要密码

class UserResponse(UserBase):
    id: int                # 响应时返回 id
    created_at: datetime

    model_config = ConfigDict(from_attributes=True)  # 支持 ORM 模型转换
```

这种继承模式是 FastAPI 最佳实践：Base → Create/Update → Response。

### 4.5 Schema 生成（AI Function Calling）

Pydantic 模型可以直接导出 JSON Schema，非常适合作为 LLM Function Calling 的工具定义：

```python
class SearchParams(BaseModel):
    """搜索互联网"""
    query: str = Field(description="搜索关键词")
    max_results: int = Field(default=5, description="最大结果数")

schema = SearchParams.model_json_schema()
# 输出标准 JSON Schema，可直接用于 OpenAI/Anthropic Function Calling
```

这是 Pydantic 在 AI 工程中最重要的应用之一。

## 5. 异步路由

### 5.1 async def vs def

```python
# 异步路由 — 适用于 I/O 密集（网络请求、数据库、文件）
@app.get("/async")
async def async_route():
    data = await httpx.AsyncClient().get("https://api.example.com")
    return data.json()

# 同步路由 — FastAPI 会自动放到线程池执行，不会阻塞事件循环
@app.get("/sync")
def sync_route():
    time.sleep(1)  # 不会阻塞其他请求
    return {"msg": "done"}
```

关键规则：
- 如果函数内部用了 `await`，**必须**用 `async def`
- 如果函数内部有阻塞操作（如同步数据库驱动），**用 `def`**（FastAPI 自动放线程池）
- **千万不要**在 `async def` 里调用阻塞操作，这会阻塞整个事件循环

### 5.2 异步数据库操作

```python
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine

engine = create_async_engine("sqlite+aiosqlite:///db.sqlite3")

async def get_db():
    async with AsyncSession(engine) as session:
        yield session

@app.get("/users")
async def list_users(db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(User))
    return result.scalars().all()
```

## 6. 后台任务

`BackgroundTasks` 用于在响应返回后执行非紧急任务（发邮件、写日志等）：

```python
from fastapi import BackgroundTasks

def send_email(to: str, subject: str):
    """耗时操作，在后台执行"""
    time.sleep(5)  # 模拟发送
    print(f"邮件已发送给 {to}")

@app.post("/register")
async def register(bg: BackgroundTasks):
    # 立即返回响应
    bg.add_task(send_email, "user@example.com", "欢迎注册")
    return {"msg": "注册成功"}
```

特点：
- 响应先返回给客户端，后台任务异步执行
- 后台任务按添加顺序**串行执行**
- 适合轻量任务；重任务应用 Celery / arq 等任务队列

## 7. 错误处理

### 7.1 HTTPException

```python
from fastapi import HTTPException

@app.get("/items/{item_id}")
async def read_item(item_id: int):
    if item_id not in db:
        raise HTTPException(
            status_code=404,
            detail="Item not found",
            headers={"X-Error": "not-found"}  # 可选的自定义 header
        )
```

### 7.2 自定义异常处理器

```python
class BusinessError(Exception):
    def __init__(self, code: str, message: str):
        self.code = code
        self.message = message

@app.exception_handler(BusinessError)
async def business_error_handler(request: Request, exc: BusinessError):
    return JSONResponse(
        status_code=400,
        content={
            "success": False,
            "error_code": exc.code,
            "message": exc.message
        }
    )
```

### 7.3 覆盖默认的 422 验证错误

```python
from fastapi.exceptions import RequestValidationError

@app.exception_handler(RequestValidationError)
async def validation_handler(request, exc):
    return JSONResponse(
        status_code=422,
        content={
            "success": False,
            "error_code": "VALIDATION_ERROR",
            "message": "请求参数校验失败",
            "details": exc.errors()
        }
    )
```

## 8. OpenAPI / Swagger 自动文档

FastAPI 自动生成 OpenAPI 3.0 规范文档：

- **Swagger UI**：`http://localhost:8000/docs`
- **ReDoc**：`http://localhost:8000/redoc`
- **OpenAPI JSON**：`http://localhost:8000/openapi.json`

自定义文档信息：

```python
app = FastAPI(
    title="My API",
    description="API 描述",
    version="1.0.0",
    docs_url="/docs",         # Swagger 路径，设为 None 可禁用
    redoc_url="/redoc",
    openapi_tags=[
        {"name": "users", "description": "用户管理"},
        {"name": "items", "description": "物品管理"},
    ]
)
```

路由的文档增强：

```python
@app.get(
    "/items/{item_id}",
    response_model=ItemResponse,     # 响应模型（自动生成 schema）
    summary="获取物品详情",
    description="根据 ID 获取物品的完整信息",
    tags=["items"],
    responses={404: {"description": "物品不存在"}}
)
```

## 9. 项目结构最佳实践

### 小型项目

```
project/
├── main.py              # FastAPI app 入口
├── models.py            # Pydantic 模型
├── database.py          # 数据库连接
└── requirements.txt
```

### 中大型项目（推荐）

```
project/
├── app/
│   ├── __init__.py
│   ├── main.py          # FastAPI app 创建、中间件注册、router 挂载
│   ├── config.py        # 配置（用 pydantic-settings）
│   ├── database.py      # 数据库引擎、session 工厂
│   ├── dependencies.py  # 公共依赖（get_db, get_current_user）
│   ├── exceptions.py    # 自定义异常 + 异常处理器
│   ├── middleware.py    # 自定义中间件
│   ├── models/          # SQLAlchemy ORM 模型
│   │   ├── user.py
│   │   └── item.py
│   ├── schemas/         # Pydantic 模型（请求/响应）
│   │   ├── user.py
│   │   └── item.py
│   ├── routers/         # 路由模块
│   │   ├── users.py
│   │   └── items.py
│   └── services/        # 业务逻辑层
│       ├── user_service.py
│       └── item_service.py
├── tests/
│   ├── conftest.py
│   ├── test_users.py
│   └── test_items.py
├── alembic/             # 数据库迁移
├── requirements.txt
└── Dockerfile
```

### 分层原则

```
Router（路由层）
  ↓ 调用
Service（业务逻辑层）
  ↓ 调用
Repository / ORM（数据访问层）
```

- **Router**：只做参数接收和响应返回，不写业务逻辑
- **Service**：业务逻辑，可被多个 Router 复用
- **Schema**（Pydantic 模型）分 `Create`、`Update`、`Response` 三类
- **依赖注入**放在 `dependencies.py`，被所有 Router 共享

## 总结

| 特性 | 核心要点 |
|------|----------|
| ASGI | 异步架构，Uvicorn 驱动，非阻塞 I/O |
| 依赖注入 | `Depends()` + `yield` 管理资源生命周期 |
| 中间件 | 洋葱模型，注意添加顺序与执行顺序相反 |
| Pydantic V2 | 类型驱动校验，`model_json_schema()` 导出给 AI |
| 异步 | `async def` 用于 I/O，`def` 自动走线程池 |
| 错误处理 | `HTTPException` + 自定义 `exception_handler` |
| 项目结构 | Router → Service → Repository 三层分离 |
