## 1. OOP：面向对象编程

### 1.1 为什么需要 OOP？

想象你在写一个银行系统。最初你用几个变量存余额、用函数来转账——这在代码量小的时候没问题。但当系统变大，你会遇到一个核心问题：**数据和操作数据的逻辑分散在各处，改一个地方要到处找关联代码**。

OOP 的本质是：**把数据和操作数据的函数绑在一起，形成一个"对象"**。这不是什么高深的哲学，而是一种管理复杂度的手段。

### 1.2 类与实例

```python
class BankAccount:
    """银行账户类"""
    
    def __init__(self, owner: str, balance: float = 0):
        self.owner = owner          # 实例属性：每个对象独立拥有
        self._balance = balance     # 下划线前缀：约定为"内部使用"
    
    def deposit(self, amount: float):
        if amount <= 0:
            raise ValueError("存款金额必须大于0")
        self._balance += amount
    
    def get_balance(self) -> float:
        return self._balance
```

**核心概念**：
- `class` 是蓝图（模板），`实例` 是根据蓝图造出来的具体对象
- `self` 指向当前实例本身。`self.owner` 就是"这个对象的 owner 属性"
- `__init__` 是初始化方法，在 `BankAccount("Alice")` 时自动调用

**常见误区**：
- `self` 不是关键字，只是约定名称（但永远不要用别的名字）
- `__init__` 不是"构造函数"——对象在调用 `__init__` 之前就已经创建了（真正的构造是 `__new__`）

### 1.3 继承

```python
class SavingsAccount(BankAccount):
    """储蓄账户，继承自银行账户"""
    
    def __init__(self, owner, balance=0, interest_rate=0.02):
        super().__init__(owner, balance)   # 调用父类的 __init__
        self.interest_rate = interest_rate
    
    def apply_interest(self):
        interest = self._balance * self.interest_rate
        self._balance += interest
        return interest
```

**为什么需要继承？** 不是为了"代码复用"（那用函数就行），而是为了表达"is-a"关系：储蓄账户**是一种**银行账户。这意味着任何接受银行账户的地方，都能接受储蓄账户——这就是**多态**。

**常见误区**：
- 不要为了复用代码而滥用继承。如果 A 和 B 只是有几个方法相同，用组合（把共用逻辑抽成独立类）而非继承
- Python 支持多重继承，但要谨慎使用，MRO（方法解析顺序）容易让人困惑

### 1.4 封装

Python 没有 Java 那种 `private` 关键字。Python 的封装靠**约定**：

| 写法 | 含义 |
|------|------|
| `name` | 公开属性，随便访问 |
| `_name` | 约定为内部使用，外部不该直接访问（但技术上可以） |
| `__name` | Name mangling，会变成 `_ClassName__name`，更难直接访问 |

**核心原理**：Python 信奉"我们都是成年人"——不强制禁止，而是通过约定告诉你"这个不该碰"。

### 1.5 实际应用场景

- **Web 框架**（Django/Flask）：Model 类映射数据库表，View 类处理请求
- **数据处理**：自定义 Pipeline 类封装数据清洗步骤
- **CLI 工具**：用类组织命令和状态

## 2. 魔术方法

### 2.1 什么是魔术方法？

魔术方法（也叫 dunder methods，因为前后都有双下划线）是 Python 的**协议**。当你写 `len(obj)` 时，Python 实际上在调用 `obj.__len__()`。这套机制让你的自定义类可以像内置类型一样使用。

### 2.2 核心魔术方法

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        """给开发者看的表示（在 REPL 和调试中显示）"""
        return f"Vector({self.x}, {self.y})"
    
    def __str__(self):
        """给用户看的表示（print 时显示）"""
        return f"({self.x}, {self.y})"
    
    def __len__(self):
        """让 len() 能工作"""
        return 2  # 二维向量
    
    def __eq__(self, other):
        """让 == 能正确比较"""
        if not isinstance(other, Vector):
            return NotImplemented
        return self.x == other.x and self.y == other.y
    
    def __add__(self, other):
        """让 + 能工作"""
        return Vector(self.x + other.x, self.y + other.y)
    
    def __getitem__(self, index):
        """让 obj[0] 能工作"""
        if index == 0: return self.x
        if index == 1: return self.y
        raise IndexError("Vector 只有两个分量")
```

### 2.3 `__repr__` vs `__str__`

这是最容易混淆的一对：

- `__repr__`：面向开发者。目标是**无歧义**，最好能 `eval(repr(obj))` 还原出对象
- `__str__`：面向用户。目标是**可读性**

**规则**：如果只实现一个，实现 `__repr__`。因为当 `__str__` 不存在时，Python 会回退到 `__repr__`。

### 2.4 常见误区

- `__eq__` 定义后，对象就不能放进 `set` 或作为 `dict` 的 key 了——除非同时定义 `__hash__`
- 返回 `NotImplemented`（不是 `raise NotImplementedError`）告诉 Python "我处理不了，让对方试试"

### 2.5 实际应用场景

- ORM 框架中，Model 实例的 `__repr__` 显示主键和关键字段
- 数据类（dataclass）自动生成 `__init__`、`__repr__`、`__eq__`
- 实现自定义容器类（如优先队列）时需要 `__len__`、`__getitem__`

## 3. 列表推导式与推导式家族

### 3.1 为什么用推导式？

推导式不只是"语法糖"——它是 Pythonic 的核心体现。对比：

```python
# 非 Pythonic
squares = []
for x in range(10):
    squares.append(x ** 2)

# Pythonic
squares = [x ** 2 for x in range(10)]
```

第二种写法不仅更短，而且**意图更明确**：一看就知道是"用某个规则生成一个列表"。而且执行速度更快（底层优化了循环）。

### 3.2 推导式家族

```python
# 列表推导式
squares = [x**2 for x in range(10)]

# 带条件
evens = [x for x in range(20) if x % 2 == 0]

# 字典推导式
word_lengths = {word: len(word) for word in ["hello", "world"]}

# 集合推导式
unique_lengths = {len(word) for word in ["hello", "world", "hi"]}

# 生成器表达式（注意是圆括号，惰性求值）
gen = (x**2 for x in range(10))
```

### 3.3 嵌套推导式

```python
# 展平二维列表
matrix = [[1, 2], [3, 4], [5, 6]]
flat = [num for row in matrix for num in row]
# 阅读顺序：先外层 for，再内层 for
```

### 3.4 常见误区

- **不要在推导式里搞复杂逻辑**。超过两层嵌套或需要 if-else 的，老老实实写 for 循环
- 生成器表达式 `(...)` 不是元组推导式——Python 没有元组推导式
- 推导式里的变量在 Python 3 中不会泄漏到外部作用域（Python 2 会）

### 3.5 实际应用场景

- 数据清洗：`clean = [s.strip().lower() for s in raw_data if s.strip()]`
- 快速构建字典：`config = {k: v for k, v in pairs if v is not None}`
- 文件过滤：`py_files = [f for f in os.listdir('.') if f.endswith('.py')]`

## 4. 生成器与 yield

### 4.1 核心问题：内存

假设你要处理一个 10GB 的日志文件。如果用列表把所有行读进内存——内存爆了。生成器的核心价值是：**一次只产生一个值，用完就扔**。

### 4.2 生成器函数

```python
def count_up(n):
    """从 1 数到 n"""
    i = 1
    while i <= n:
        yield i      # 暂停函数，返回 i，下次调用时从这里继续
        i += 1

# 使用
for num in count_up(5):
    print(num)  # 1, 2, 3, 4, 5
```

**核心原理**：
- `yield` 让函数变成生成器函数
- 调用生成器函数不会执行函数体，而是返回一个生成器对象
- 每次 `next()` 或 `for` 迭代时，函数从上次 `yield` 的位置继续执行

### 4.3 yield 的本质

普通函数是"调用-返回"模型：调用时从头开始，return 后就结束了。

生成器是"暂停-恢复"模型：`yield` 让函数"冻结"在当前状态，下次被唤醒时从冻结点继续。

```python
def read_large_file(filepath):
    """逐行读取大文件，不会占用大量内存"""
    with open(filepath) as f:
        for line in f:
            yield line.strip()

# 即使文件有 10GB，内存中也只有一行
for line in read_large_file("huge.log"):
    process(line)
```

### 4.4 yield from

```python
def chain(*iterables):
    for it in iterables:
        yield from it   # 等价于 for item in it: yield item

list(chain([1, 2], [3, 4]))  # [1, 2, 3, 4]
```

### 4.5 常见误区

- 生成器只能遍历一次。遍历完后再 `for` 不会报错，只是没有输出
- 生成器是惰性的，如果你在 `yield` 前有副作用（如打印），调用函数时不会执行

### 4.6 实际应用场景

- 大文件逐行处理
- 数据管道：多个生成器串联，每一步只处理一个元素
- 无限序列（如斐波那契数列、传感器数据流）
- `itertools` 模块大量使用生成器

## 5. 装饰器（Decorator）

### 5.1 前置知识：闭包

要理解装饰器，必须先理解闭包。

```python
def make_greeter(greeting):
    def greet(name):
        return f"{greeting}, {name}!"   # greeting 来自外层函数
    return greet

hello = make_greeter("Hello")
hello("Alice")  # "Hello, Alice!"
```

**闭包**：内层函数引用了外层函数的变量，并且外层函数已经返回了。内层函数"记住"了外层变量——这就是闭包。

### 5.2 装饰器的本质

装饰器就是一个**接收函数、返回函数**的函数。

```python
def timer(func):
    """计时装饰器"""
    import time
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        elapsed = time.time() - start
        print(f"{func.__name__} 耗时 {elapsed:.4f}s")
        return result
    return wrapper

@timer                        # 语法糖，等价于 slow_func = timer(slow_func)
def slow_func():
    import time
    time.sleep(1)
```

### 5.3 @ 语法糖

```python
@timer
def my_func():
    pass

# 完全等价于：
def my_func():
    pass
my_func = timer(my_func)
```

`@` 不是什么神秘的东西，就是把下面的函数作为参数传给装饰器，然后用返回值替换原函数。

### 5.4 functools.wraps

```python
from functools import wraps

def timer(func):
    @wraps(func)         # 保留原函数的 __name__、__doc__ 等属性
    def wrapper(*args, **kwargs):
        # ...
        return func(*args, **kwargs)
    return wrapper
```

**不加 `@wraps` 会怎样？** `my_func.__name__` 会变成 `"wrapper"`，调试时会很困惑。

### 5.5 带参数的装饰器

```python
def repeat(n):
    """执行 n 次的装饰器"""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)     # repeat(3) 返回 decorator，decorator 再装饰函数
def say_hello():
    print("Hello!")
```

三层嵌套看着复杂，但逻辑很清楚：最外层接收装饰器参数，中间层接收被装饰的函数，最内层是实际执行的 wrapper。

### 5.6 常见误区

- 装饰器在**模块导入时**就执行了（`@timer` 那行），不是在函数调用时
- 多个装饰器的执行顺序是**从下往上**装饰，从上往下执行

### 5.7 实际应用场景

- **Web 框架**：`@app.route("/api/users")` 注册路由
- **权限控制**：`@login_required` 检查登录状态
- **缓存**：`@lru_cache` 缓存函数结果
- **重试**：`@retry(max_attempts=3)` 失败自动重试
- **日志**：自动记录函数调用参数和返回值

## 6. 迭代器协议

### 6.1 迭代器是什么？

当你写 `for item in something` 时，Python 做了什么？

1. 调用 `iter(something)` 得到一个迭代器
2. 反复调用 `next(iterator)` 获取下一个值
3. 遇到 `StopIteration` 异常时停止

所以，迭代器就是实现了 `__iter__()` 和 `__next__()` 两个方法的对象。

### 6.2 可迭代对象 vs 迭代器

```python
my_list = [1, 2, 3]        # 可迭代对象（有 __iter__）
my_iter = iter(my_list)     # 迭代器（有 __iter__ 和 __next__）

next(my_iter)  # 1
next(my_iter)  # 2
next(my_iter)  # 3
next(my_iter)  # StopIteration!
```

**关键区别**：列表是可迭代对象，可以反复遍历。迭代器是一次性的，遍历完就空了。

### 6.3 自定义迭代器

```python
class Countdown:
    def __init__(self, start):
        self.current = start
    
    def __iter__(self):
        return self
    
    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        self.current -= 1
        return self.current + 1

for num in Countdown(5):
    print(num)  # 5, 4, 3, 2, 1
```

### 6.4 为什么要理解迭代器？

因为 Python 的 `for` 循环、推导式、`*` 解包、`zip()`、`map()` 等等，底层全部基于迭代器协议。理解它，你就理解了 Python 数据流动的基础。

### 6.5 常见误区

- 文件对象本身就是迭代器（`for line in open("file")` 之所以能工作）
- `range()` 不是迭代器，是可迭代对象。每次 `for` 都会创建新的迭代器

## 7. 上下文管理器与 with

### 7.1 解决什么问题？

```python
# 不用 with（有风险）
f = open("file.txt")
data = f.read()
f.close()       # 如果 f.read() 抛异常，这行不会执行！
```

`with` 语句保证：**不管代码块里发生什么（包括异常），资源一定会被正确释放**。

```python
# 用 with（安全）
with open("file.txt") as f:
    data = f.read()
# 离开 with 块时，f.close() 自动调用
```

### 7.2 上下文管理器协议

实现 `__enter__` 和 `__exit__` 两个方法：

```python
class DatabaseConnection:
    def __enter__(self):
        self.conn = create_connection()
        return self.conn
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        self.conn.close()
        return False   # False = 不吞掉异常，让它继续传播

with DatabaseConnection() as conn:
    conn.execute("SELECT ...")
```

### 7.3 contextlib：更简洁的写法

```python
from contextlib import contextmanager

@contextmanager
def timer(label):
    import time
    start = time.time()
    yield                      # yield 之前是 __enter__，之后是 __exit__
    elapsed = time.time() - start
    print(f"{label}: {elapsed:.4f}s")

with timer("数据处理"):
    heavy_computation()
```

### 7.4 常见误区

- `__exit__` 返回 `True` 会**吞掉异常**——通常你不想这样做
- `with` 不创建新的作用域，`as` 后面的变量在 `with` 块外面依然可用

### 7.5 实际应用场景

- 文件操作（最常见）
- 数据库连接/事务
- 锁（`threading.Lock`）
- 临时修改配置，结束后恢复
- 测试中的 mock/patch

## 8. Type Hints（类型提示）

### 8.1 Python 不是强类型吗？

Python 是**强类型**（`"1" + 1` 会报错）但**动态类型**（变量不需要声明类型）。Type Hints 是 Python 3.5+ 引入的**可选**类型标注，不影响运行时行为。

### 8.2 为什么要用？

```python
# 没有 Type Hints
def process(data, flag):
    ...

# 有 Type Hints
def process(data: list[dict[str, Any]], flag: bool = False) -> str:
    ...
```

第二种写法，不用看函数体就知道参数和返回值的类型。好处：
1. **代码即文档**：IDE 和人都能快速理解接口
2. **IDE 自动补全**：知道类型后，`.` 后面的提示更准确
3. **静态检查**：`mypy` 可以在运行前发现类型错误

### 8.3 常用语法

```python
from typing import Optional, Union

# 基本类型
name: str = "Alice"
age: int = 30
scores: list[float] = [90.5, 85.0]
config: dict[str, Any] = {"debug": True}

# 函数签名
def greet(name: str, times: int = 1) -> str:
    return f"Hello {name}! " * times

# Optional = 可能是 None
def find_user(user_id: int) -> Optional[User]:
    ...

# Union = 多种类型
def process(data: Union[str, bytes]) -> str:
    ...

# Python 3.10+ 可以用 | 代替 Union
def process(data: str | bytes) -> str:
    ...
```

### 8.4 常见误区

- Type Hints **不影响运行时**。标注了 `int` 你传 `str` 也不会报错（除非用 mypy 检查）
- 不要过度标注。局部变量类型显而易见的就不用标注
- `list[int]` 在 Python 3.9+ 才能直接用，之前需要 `from typing import List` 用 `List[int]`

### 8.5 实际应用场景

- API 接口定义（FastAPI 强依赖 Type Hints）
- 团队协作（类型标注是最好的文档）
- 大型项目（mypy 静态检查防止类型相关 bug）

## 9. 常用标准库

### 9.1 os 和 sys

**os**：与操作系统交互（文件、目录、环境变量）

```python
import os

os.getcwd()                          # 当前工作目录
os.listdir(".")                      # 列出目录内容
os.path.join("dir", "file.txt")      # 拼接路径（跨平台安全）
os.path.exists("file.txt")           # 文件是否存在
os.makedirs("a/b/c", exist_ok=True)  # 递归创建目录
os.environ.get("HOME")               # 读取环境变量
```

**sys**：与 Python 解释器交互

```python
import sys

sys.argv          # 命令行参数列表
sys.path          # 模块搜索路径
sys.version       # Python 版本
sys.exit(0)       # 退出程序
sys.stdin/stdout  # 标准输入输出
```

**现代替代**：`pathlib.Path` 是 os.path 的现代替代（Python 3.4+），面向对象风格更直观：

```python
from pathlib import Path
p = Path("data") / "file.txt"   # 用 / 拼接路径
p.exists()
p.read_text()
p.mkdir(parents=True, exist_ok=True)
```

### 9.2 datetime

```python
from datetime import datetime, timedelta, date

now = datetime.now()
today = date.today()

# 格式化
now.strftime("%Y-%m-%d %H:%M:%S")   # "2026-04-09 14:30:00"

# 解析
dt = datetime.strptime("2026-04-09", "%Y-%m-%d")

# 时间运算
tomorrow = today + timedelta(days=1)
diff = datetime(2026, 12, 31) - datetime.now()
print(f"距离年底还有 {diff.days} 天")
```

**常见误区**：
- `datetime.now()` 返回的是**本地时间**，没有时区信息。处理跨时区要用 `datetime.now(timezone.utc)`
- 时间比较要确保两边都有/都没有时区信息，不能混用

### 9.3 json

```python
import json

# Python 对象 → JSON 字符串
data = {"name": "Alice", "age": 30, "scores": [90, 85]}
json_str = json.dumps(data, ensure_ascii=False, indent=2)

# JSON 字符串 → Python 对象
obj = json.loads(json_str)

# 读写文件
with open("data.json", "w") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

with open("data.json") as f:
    data = json.load(f)
```

**`ensure_ascii=False`**：默认 json.dumps 会把中文转成 `\uXXXX`，加这个参数保留中文。

### 9.4 collections

```python
from collections import Counter, defaultdict, namedtuple, deque

# Counter：计数器
words = ["apple", "banana", "apple", "cherry", "apple"]
count = Counter(words)       # Counter({'apple': 3, 'banana': 1, 'cherry': 1})
count.most_common(2)         # [('apple', 3), ('banana', 1)]

# defaultdict：带默认值的字典
word_index = defaultdict(list)
for i, word in enumerate(words):
    word_index[word].append(i)   # 不需要先检查 key 是否存在

# namedtuple：具名元组（轻量级的不可变类）
Point = namedtuple("Point", ["x", "y"])
p = Point(3, 4)
print(p.x, p.y)    # 用名字访问，比 p[0] 清晰

# deque：双端队列（两端操作都是 O(1)）
dq = deque([1, 2, 3])
dq.appendleft(0)    # [0, 1, 2, 3]
dq.pop()             # 3
```

### 9.5 itertools

```python
from itertools import chain, product, combinations, permutations, groupby, islice

# chain：串联多个可迭代对象
list(chain([1, 2], [3, 4]))  # [1, 2, 3, 4]

# product：笛卡尔积
list(product("AB", "12"))    # [('A','1'), ('A','2'), ('B','1'), ('B','2')]

# combinations：组合
list(combinations("ABC", 2)) # [('A','B'), ('A','C'), ('B','C')]

# islice：对迭代器切片（不用全部加载到内存）
from itertools import count
list(islice(count(1), 5))    # [1, 2, 3, 4, 5]
```

### 9.6 re（正则表达式）

```python
import re

# 查找
re.search(r'\d+', 'age: 30')          # Match 对象, .group() = '30'
re.findall(r'\d+', 'a1b2c3')          # ['1', '2', '3']

# 替换
re.sub(r'\s+', ' ', 'a  b   c')       # 'a b c'

# 分组
m = re.match(r'(\d{4})-(\d{2})-(\d{2})', '2026-04-09')
m.group(1)  # '2026'
m.group(2)  # '04'

# 编译（多次使用同一正则时提升性能）
pattern = re.compile(r'\b\w+@\w+\.\w+\b')
emails = pattern.findall(text)
```

**常见误区**：
- 用 `r''` 原始字符串避免转义问题（`r'\n'` 是两个字符，`'\n'` 是换行符）
- `re.match` 只匹配字符串**开头**，`re.search` 匹配**任意位置**
- 正则表达式能力强大但可读性差，简单的字符串操作优先用 `str` 方法

## 10. 包管理与虚拟环境

### 10.1 为什么需要虚拟环境？

项目 A 需要 `requests==2.25`，项目 B 需要 `requests==2.31`。如果装在全局，两个项目至少有一个会挂。**虚拟环境让每个项目有独立的依赖空间**。

### 10.2 venv（Python 内置）

```bash
# 创建虚拟环境
python3 -m venv .venv

# 激活
source .venv/bin/activate      # macOS/Linux
.venv\Scripts\activate         # Windows

# 安装依赖
pip install requests flask

# 导出依赖
pip freeze > requirements.txt

# 从 requirements.txt 安装
pip install -r requirements.txt

# 退出虚拟环境
deactivate
```

### 10.3 requirements.txt

```
# 精确版本（推荐用于部署）
requests==2.31.0
flask==3.0.0

# 版本范围（用于库开发）
requests>=2.25,<3.0
```

### 10.4 conda

```bash
# 创建环境
conda create -n myenv python=3.11

# 激活
conda activate myenv

# 安装
conda install numpy pandas

# 导出
conda env export > environment.yml
```

**venv vs conda**：
- venv：Python 内置，轻量，只管 Python 包
- conda：独立工具，能管理非 Python 依赖（如 C 库），科学计算首选

### 10.5 poetry（现代方案）

```bash
# 初始化项目
poetry init

# 添加依赖
poetry add requests
poetry add --group dev pytest

# 安装所有依赖
poetry install

poetry run python main.py
```

Poetry 用 `pyproject.toml` 替代 `requirements.txt` + `setup.py`，是更现代的方案。

## 11. 调试技巧

### 11.1 print 大法

最简单但也最有效：

```python
def process(data):
    print(f"[DEBUG] data type: {type(data)}, len: {len(data)}")
    print(f"[DEBUG] data[:3] = {data[:3]}")
    result = transform(data)
    print(f"[DEBUG] result = {result}")
    return result
```

**升级版**：用 f-string 的 `=` 语法（Python 3.8+）：

```python
x = 42
print(f"{x=}")  # 输出: x=42
```

### 11.2 pdb（Python Debugger）

```python
# 在代码中插入断点
import pdb; pdb.set_trace()

# Python 3.7+ 更简洁
breakpoint()
```

pdb 常用命令：
| 命令 | 说明 |
|------|------|
| `n` (next) | 执行下一行 |
| `s` (step) | 进入函数内部 |
| `c` (continue) | 继续运行到下一个断点 |
| `p expr` | 打印表达式的值 |
| `l` (list) | 显示当前位置附近的代码 |
| `q` (quit) | 退出调试器 |

### 11.3 IDE 断点

VS Code / PyCharm 中，点击行号左侧设置断点，然后 Debug 运行。相比 pdb：
- 可以直观看到所有变量的值
- 可以设置条件断点（如"只在 i > 100 时停下"）
- 可以查看调用栈

### 11.4 实用建议

- 先用 `print` 快速定位问题范围
- 缩小到具体函数后，用 `breakpoint()` 深入
- 复杂场景用 IDE 断点 + Watch 表达式
- 日志比 print 更好：`import logging`，可以控制级别、输出到文件

## 12. Git 基础

### 12.1 为什么用 Git？

Git 不只是"代码备份"——它是**时间机器 + 平行宇宙管理器**。你可以：
- 随时回到任何历史版本
- 在不同分支上同时开发不同功能
- 多人协作不会互相覆盖代码

### 12.2 核心概念

```
工作区 (Working Directory)
   ↓  git add
暂存区 (Staging Area)
   ↓  git commit
本地仓库 (Local Repository)
   ↓  git push
远程仓库 (Remote Repository)
```

### 12.3 常用命令

```bash
# 初始化
git init
git clone <url>

# 日常操作
git status              # 查看状态
git add file.py         # 添加到暂存区
git add .               # 添加所有修改
git commit -m "message" # 提交
git push                # 推送到远程

# 查看历史
git log --oneline       # 简洁日志
git diff                # 查看未暂存的修改
git diff --staged       # 查看已暂存的修改

# 分支
git branch feature-x    # 创建分支
git checkout feature-x  # 切换分支（或 git switch feature-x）
git merge feature-x     # 合并分支

# 撤销
git checkout -- file.py # 丢弃工作区修改
git reset HEAD file.py  # 取消暂存
git revert <commit>     # 创建一个反向提交来撤销某次提交
```

### 12.4 .gitignore

```
# .gitignore 文件
__pycache__/
*.pyc
.env
.venv/
node_modules/
.DS_Store
*.log
```

**为什么需要？** 不是所有文件都该提交：虚拟环境、编译产物、密钥文件、IDE 配置——这些不该进仓库。

### 12.5 Commit Message 规范

```
feat: 添加用户注册功能
fix: 修复登录页面白屏问题
docs: 更新 API 文档
refactor: 重构数据处理模块
test: 添加用户模块单元测试
chore: 更新依赖版本
```

### 12.6 实际工作流

```bash
# 1. 开始新功能
git checkout -b feature/user-auth

# 2. 开发过程中多次提交
git add .
git commit -m "feat: 添加登录页面"
git commit -m "feat: 添加 JWT 验证"

# 3. 完成后合并
git checkout main
git merge feature/user-auth

# 4. 推送
git push origin main
```

### 12.7 常见误区

- `git pull` = `git fetch` + `git merge`，可能产生合并冲突
- 不要把大文件（视频、数据集）放进 Git，用 Git LFS
- commit 要小而频繁，不要攒一大堆改动一次性提交
