## 1. Why：没有观测 = 黑盒

用户反馈「答错了」时，需回答：**检索命中了吗？调了哪个工具？模型原始 reasoning？耗时在哪一步？** 没有 trace 只能猜。

---

## 2. Logs：结构化与关联 ID

- **JSON 一行一条**，字段含 `timestamp`、`level`、`trace_id`、`span_id`、`message`。  
- **`contextvars`** 在 asyncio 下传递 `trace_id`（见 `demos/01_trace_contextvars.py`）。  
- **禁止**：完整 prompt、PII、密钥入日志——用 **哈希或截断**。

---

## 3. Metrics：SLI 与告警

- **延迟**：P50/P95 **首 token**、**端到端**。  
- **错误率**：HTTP 5xx、工具失败、解析失败。  
- **业务**：检索空率、工具调用次数分布、**每会话 Token/费用**。  
- **告警**：错误率突增、延迟 SLO 破位、队列积压。

---

## 4. Traces：Span 划分

典型 Span：`http.request` → `agent.route` → `retrieve` → `llm` → `tool.http` → `llm` → `response.serialize`。  
**父子关系**表达嵌套；**并行检索**用多个子 span + `links`。

---

## 5. Agent 特有字段

- **检索**：`query`、`top_k`、命中 `doc_id`、分数、**是否 rerank**。  
- **工具**：`name`、**脱敏后参数**、耗时、重试次数。  
- **模型**：`model_id`、**prompt_tokens** / **completion_tokens**。

---

## 6. 隐私与采样

- **头部采样**（仅记录 1% trace）降低成本，但 **错误全量** 捕获。  
- **分级**：开发全量、生产采样 + **动态提升采样率**（某用户报错时）。

---

## 7. 与测试、发布的衔接

- **契约测试**：工具与路由（A5-测试与CI）。  
- **金丝雀 / 特性开关**：新 prompt 与旧版对比 **同 trace 维度**。  
- **回放**：用固定 seed + **录制的 tool mock** 做回归（注意脱敏）。
