## 1. 导读：怎么答技术面试题

### Q：技术题要答到什么程度？

**答：**  
- **30 秒版**：一句话结论 + 一个关键词（例如「用 asyncio 重叠 IO 等待」）。  
- **2 分钟版**：加「适用边界」+「反面什么时候不用」+「我们项目里……」。  
- **系统设计**：先澄清需求（QPS、延迟、数据规模、合规），再分层（网关 → 应用 → 模型 → 向量库 → 任务队列），最后谈**失败模式**与**演进**。

### Q：Agent 岗常考的「能力模型」是什么？

**答：** 通常拆四块：**模型与 Prompt**、**检索与数据**、**工具与编排**、**工程与线上**（可靠、安全、观测、成本）。缺哪块就补哪块的短答案。

## 2. Python 语言与运行时

### Q：解释 GIL；对多线程 / 多进程 / asyncio 各有什么影响？

**答：** GIL 是 CPython 的全局解释器锁，**同一时刻只有一个线程执行 Python 字节码**。  
- **CPU 密集**：多线程**不能**并行多核；用 **multiprocessing** 或把热点放到释放 GIL 的 C 扩展（如 NumPy 部分运算）。  
- **IO 密集**：线程在阻塞 IO 时会释放 GIL，**多线程或 asyncio** 都能重叠等待；asyncio 在大量连接时更省线程开销。  
- **asyncio**：单线程协作式多任务，**禁止**在协程里长时间占用 CPU 或阻塞 IO，否则卡死事件循环；阻塞调用应 **`run_in_executor`** 或换异步库。

### Q：`is` 和 `==` 区别？`None` 比较用什么？

**答：** `is` 比对象身份（同一内存对象），`==` 比值（可自定义 `__eq__`）。**`None` 用 `is None`**，不要用 `== None`。

### Q：可变默认参数有什么坑？

**答：** `def f(x, items=[])` 里 `items` 在函数定义时创建一次，**多次调用共享同一列表**。默认值用 **`None` 再在函数体内 `if items is None: items = []`**。

### Q：`*args` 和 `**kwargs`？仅限关键字参数怎么写？

**答：** `*args` 收多余位置参数为元组，`**kwargs` 收关键字为字典。Python 3 可用 `*, kw_only` 强制后续参数必须关键字传入。

### Q：装饰器在 Agent 代码里常见用途？

**答：** 日志、重试、鉴权、计时、限流——横切逻辑。注意 Python 的 `@decorator` 与 GoF「装饰器模式」名字撞车，面试可说清「语法糖 + 闭包」即可。

### Q：`typing` 里 `List` vs `list`（3.9+）？

**答：** 3.9+ 内建泛型 **`list[str]`** 即可；旧代码常见 `from typing import List`。`Optional[X]` 等价 `X | None`（3.10+）。

### Q：浅拷贝与深拷贝？

**答：** 浅拷贝复制**最外层容器**，内层可变对象仍共享引用；深拷贝 **递归复制**。Agent 里拷贝 `messages` 列表时要注意嵌套 dict。

### Q：生成器与迭代器的好处？

**答：** **惰性求值**，省内存；适合流式读大文件、流式处理 token。缺点是只能单向遍历一次（除非重新创建生成器）。

## 3. Python 工程化与协作

### Q：`venv` / `poetry` / `uv` 你用什么？

**答：** 先说团队规范。**venv + pip + requirements.txt** 最通用；**poetry / uv** 解决依赖解析与锁文件、可复现构建。生产要**锁版本**、分层 **requirements-prod.txt / dev**。

### Q：`pyproject.toml` 里常见字段？

**答：** 项目元数据、构建系统（PEP 517）、**可选依赖组**、工具配置（black/ruff/pytest）。面试提「单一真源」即可。

### Q：如何做代码质量与 CI？

**答：** **ruff/flake8 + black（或 ruff format）+ mypy（按需）+ pytest**；PR 上跑 **GitHub Actions / GitLab CI**；关键路径**覆盖率**门槛。Agent 项目重点测：**工具参数解析、路由、RAG 检索契约**（mock LLM）。

### Q：结构化日志为什么重要？

**答：** JSON 日志便于 **ELK/Loki** 查询；带 **`trace_id` / `request_id`** 才能串 Agent 多步调用。不要打印密钥与用户全文 PII。

## 4. Web 后端、HTTP 与流式

### Q：REST 里 GET/POST 幂等性？

**答：** **GET/PUT/DELETE** 设计为幂等（重复执行结果一致）；**POST** 默认不幂等（重复可能创建多条）。Agent 写接口应用 **Idempotency-Key** 防重。

### Q：401 和 403？

**答：** **401** 未认证或 Token 无效；**403** 已认证但无权限。

### Q：502 / 503 / 504？

**答：** **502** 网关从上游拿到非法响应；**503** 服务过载/维护主动不可用；**504** 网关等上游**超时**。

### Q：SSE 与 WebSocket 选型？

**答：** **SSE**：HTTP 单向、服务端推文本流，浏览器 `EventSource`，**穿透代理需注意缓冲**（nginx `proxy_buffering off` 等）。**WebSocket**：全双工、二进制友好，适合协同与长连接。**LLM 流式输出**常见 SSE 或 chunked JSON lines。

### Q：FastAPI 相对 Flask 的优势（Agent 场景）？

**答：** 原生 **async**、自动 **OpenAPI**、**Pydantic** 校验与依赖注入 `Depends`，适合高并发 IO 与类型清晰的 API。缺点：团队要会写异步与排查事件循环阻塞。

### Q：CORS 错误为什么 Postman 正常、浏览器报错？

**答：** CORS 是**浏览器**策略；服务端需正确返回 **`Access-Control-Allow-Origin`** 等，预检 **OPTIONS** 要处理。

## 5. 数据存储、缓存与消息

### Q：PostgreSQL 事务隔离级别大致区别？

**答：** 从弱到强：**读已提交**（常见默认）→ **可重复读**（同事务内读一致）→ **串行化**。Agent 写业务表要想清**幻读**与**长事务**锁表问题。

### Q：向量数据库与关系库分工？

**答：** 关系库管**强一致业务状态**（订单、用户、权限）；向量库管**语义检索**（文档块 embedding）。元数据（文档 id、权限标签）应能**过滤**，不能只靠余弦相似度。

### Q：Redis 在 Agent 系统里常见用途？

**答：** **缓存**（prompt 结果、检索结果）、**限流计数**、**会话**、**任务队列**（List/Stream）、**分布式锁**（需谨慎 TTL 与续期）。注意**持久化策略**与**丢数据**风险。

### Q：消息队列解决什么问题？

**答：** **削峰**、**解耦**、**异步长任务**（索引构建、批量 embedding）。消费端要 **幂等**（至少一次投递会重复）。

## 6. 并发、异步与任务系统

### Q：`asyncio.gather` 与 `create_task`？

**答：** `gather` 等待多个 awaitable；`create_task` 立即调度协程任务。要注意**异常处理**（`return_exceptions=True`）与**取消**（`Task.cancel()`）。

### Q：Celery / RQ 使用场景？

**答：** **CPU/长耗时/不可靠网络** 的后台任务，避免阻塞 HTTP。要配置 **broker、重试、死信、监控**。

### Q：线程池在 asyncio 里何时用？

**答：** **阻塞的第三方 SDK**（无 async 版）用 **`loop.run_in_executor`**，避免阻塞事件循环。

## 7. LLM 基础与推理

### Q：Transformer 自注意力在直觉上做什么？

**答：** 每个 token 对序列中其他 token 做**加权聚合**，权重由 QK 相似度得到，从而捕获**长距离依赖**。Decoder-only 模型用**因果掩码**保证只看过去。

### Q：Tokenizer 与 API 计费关系？

**答：** API 通常按 **token** 计费；**中英文 token 密度不同**。要在产品侧做 **token 预估与预算**。

### Q：Temperature、Top-p、Top-k？

**答：** **Temperature** 调节分布平滑度：低更确定，高更随机。**Top-p** 从累积概率截断候选集；**Top-k** 只保留概率最高的 k 个。任务越需**事实性**，温度通常越低。

### Q：什么是 KV Cache？为什么上下文长更吃显存？

**答：** 自回归生成时，已生成 token 的 K/V 可缓存，避免每步重复计算。**上下文越长，cache 越大**，推理显存与首 token 延迟上升。

### Q：上下文窗口限制下你怎么做？

**答：** **截断策略**（保留 system + 最近轮次）、**摘要压缩**、**RAG 只塞相关块**、**多段调用**（map-reduce）、**换长上下文模型**（有成本）。

### Q：Fine-tuning vs RAG vs Prompt？

**答：**  
- **Prompt**：最快，改指令与示例。  
- **RAG**：注入**可更新事实**与私域文档。  
- **Fine-tuning**：改**风格、格式、领域术语**，成本高、迭代慢。  
常组合：**FT 风格 + RAG 事实**。

## 8. Prompt 与上下文工程

### Q：System / User / Assistant 消息各干什么？

**答：** **System**：稳定规则、角色、工具使用约束；**User**：任务输入；**Assistant**：模型历史回复（含 tool_calls）。改 system 影响全局行为。

### Q：Few-shot 什么时候有效？

**答：** 需要**固定输出格式**或**分类边界**时；shot 要**多样、代表边界情况**，避免过拟合少数模板。

### Q：如何避免指令被用户注入覆盖？

**答：** **结构化包装用户输入**（XML/JSON 块）、明确「用户内容在以下块内」、**权限与工具白名单**在服务端强制执行，**不能**只靠 prompt 说「不要越狱」。

## 9. RAG 与知识库

### Q：RAG 基本流水线？

**答：** 加载文档 → **分块** → **嵌入** → 存向量库 → 查询时 **query 嵌入** → **Top-K 检索** → **拼进 prompt** → LLM 生成。可选 **重排序（rerank）**、**混合检索（BM25+向量）**。

### Q：分块策略要考虑什么？

**答：** **chunk 大小**（256–1024 tokens 常见）、**重叠**（防断句）、**按标题/段落/表格**语义边界、**元数据**（来源、时间、权限）。

### Q：为什么需要 hybrid search？

**答：** 纯向量对**专有名词、ID、错拼**可能弱；**BM25 关键词**互补。融合方式：**加权分数、RRF** 等。

### Q：Reranker 做什么？

**答：** 对候选片段做**精排**（交叉编码器等），**召回宽、排序准**，常是性价比最高的精度提升手段之一。

### Q：如何评测 RAG？

**答：** **检索**：Hit Rate、MRR、nDCG；**生成**：人工评分、LLM-as-judge（需谨慎）、**引用命中率**（答案是否被上下文支持）。要有**固定测试集**回归。

### Q：幻觉在 RAG 里怎么降？

**答：** 强制**引用片段**、**无则拒答**、**温度降低**、**rerank**、**过滤低相似度**、对答案做**二次校验**（self-check）。

### Q：Graph RAG 适用场景？

**答：** **多跳关系**、实体密集领域（法律、医疗等）；成本高，**不是默认选项**。

## 10. Function Calling、工具与 MCP

### Q：Function Calling 流程（服务端视角）？

**答：** 向模型提供 **tools 定义（JSON Schema）** → 模型返回 **tool_calls** → **服务端校验并执行**真实函数 → 将 **tool 结果消息**写回对话 → 模型继续生成最终回复。**执行权在服务端**，模型只提议。

### Q：工具 Schema 写好要注意什么？

**答：** **description 写清副作用与约束**；**required** 明确；用 **enum** 限制取值；避免过大 schema 占满上下文。

### Q：如何防止危险工具被调用？

**答：** **白名单**工具名、**参数校验**、**权限与租户隔离**、**只读/读写分级**、**网络 egress 限制**、**人工确认**（高风险操作）。

### Q：MCP 解决什么问题？

**答：** **工具与上下文的标准化交付**，同一 MCP Server 可被多个宿主复用，减少「每个框架写一套适配器」。传输有 **stdio / 远程** 等，要关注**安全边界**。

### Q：结构化输出（JSON mode）与 Function Calling 区别？

**答：** JSON mode 偏向**固定字段抽取/分类**；Function Calling 偏向**动作 + 参数**。生产都要 **parse 失败重试** 与 **降级**。

## 11. Agent 架构与单 Agent

### Q：什么是 ReAct 式循环？

**答：** **推理 →（可选）行动（工具）→ 观察 → 再推理**，直到得出答案或触达停止条件。实现上是带 **消息历史** 的多轮调用。

### Q：Agent 何时会死循环？

**答：** 模型反复调用同一工具、无法识别任务完成、或观察信息无效。需要 **max_iterations**、**重复检测**、**超时**、**费用上限**。

### Q：Agent 记忆怎么设计？

**答：** **短期**：对话 buffer；**摘要**：滚动压缩；**长期**：向量库/数据库存用户事实，**按查询检索**注入。注意 **隐私与遗忘策略**。

### Q：规划（Planner）和执行（Executor）要拆分吗？

**答：** 复杂任务可拆：**规划**产出步骤或子目标，**执行**逐步调工具。简单任务单循环即可，过度拆分增加延迟与失败点。

## 12. 多 Agent 与工作流

### Q：为什么需要多 Agent？

**答：** **上下文隔离**（不同 system 与工具集）、**专长分工**、**并行**（多检索源）。代价：**编排复杂、成本与延迟上升、调试难**。

### Q：常见多 Agent 拓扑？

**答：** **Router/Supervisor** 分发；**Pipeline** 阶段串联；**Peer 讨论**（投票/辩论，慎用终止条件）。

### Q：多 Agent 如何调试？

**答：** 统一 **trace_id**、每步记录 **输入输出摘要**、工具调用结构化日志；对同一用例做 **replay**。

## 13. 可靠性、性能与成本

### Q：调用 LLM API 如何设计超时与重试？

**答：** **连接超时 + 读超时**；**指数退避 + 抖动**；**仅对幂等或可去重**的写操作重试；**429** 遵守 **Retry-After**。流式要单独设计**读空闲超时**。

### Q：幂等键用在哪？

**答：** 用户重复点击、网关重试导致**重复下单/重复扣款**；用 **Idempotency-Key** 在服务端缓存结果一段时间。

### Q：熔断与降级？

**答：** 下游持续失败时 **快速失败** 避免拖垮线程池；**降级**为缓存结果、简化回答或切换备用模型。

### Q：如何做限流？

**答：** **令牌桶/漏桶**；按 **用户/API Key/租户**；模型侧还有 **TPM/RPM** 配额，要 **队列化** 或 **排队提示**。

### Q：成本优化手段？

**答：** 小模型路由简单任务、**缓存**重复 query、**prompt 压缩**、**批处理 embedding**、**流式改善体验但不减 token**、商用 **prompt cache**、监控 **每会话成本**。

## 14. 安全、合规与对抗

### Q：Prompt Injection 是什么？怎么防？

**答：** 用户通过输入**操纵模型**忽略原指令或泄露 system prompt。**防**：**权限在服务端**、工具白名单、**最小权限**、敏感操作二次确认、**输出过滤**、RAG 文档**不可信**当用户输入处理。

### Q：RAG 投毒？

**答：** 恶意文档进入知识库导致**错误检索与有害建议**。**防**：**文档准入与审核**、权限隔离、**来源可信度**、监控异常检索。

### Q：PII 与日志？

**答：** **最小化采集**、脱敏/哈希、**分级留存**；合规（GDPR 等）下支持删除与导出。

## 15. 可观测性、测试与发布

### Q：Agent 系统要监控什么指标？

**答：** **延迟**（首 token、整段）、**成功率/错误码**、**每步 tool 成功率**、**检索命中率**、**Token 与费用**、**队列积压**。

### Q：Tracing 要覆盖哪些 Span？

**答：** HTTP 入口 → **路由/意图** → **检索** → **每次 LLM** → **每次工具** → **后处理**。关联同一 **trace_id**。

### Q：如何测 Agent？

**答：** **契约测试**（工具参数解析）、**黄金用例集**（回归）、**mock LLM** 固定输出测分支、对 **RAG** 用固定 corpus 测检索指标。

### Q：上线策略？

**答：** **灰度**、**特性开关**、**影子流量**（只记录不返回）、**回滚**与 **kill switch**（关工具/关 Agent）。

## 16. 系统设计综合题

### Q：设计一个企业文档问答（RAG）系统。

**答（要点）：**  
- **需求**：文档格式、更新频率、QPS、延迟、权限模型、是否要多轮。  
- **写入路径**：上传 → 解析 → 分块 → 嵌入 → 向量库 + **元数据索引**（租户/标签）。  
- **读取路径**：鉴权 → query 改写（可选）→ 混合检索 → rerank → **上下文裁剪** → LLM → **引用**。  
- **失败**：解析失败、嵌入失败、检索空、模型超时；**异步任务**与 **重试**。  
- **演进**：人工反馈闭环、评测平台、多区域部署。

### Q：设计一个带工具的客服 Agent。

**答：**  
- **工具**：订单查询（只读）、工单创建（写）、知识库检索。  
- **编排**：ReAct + **max steps**；高风险操作 **确认流**。  
- **会话**：短期 memory + **业务系统**为真源。  
- **观测**：每轮 tool 与 latency；**敏感词与 PII** 过滤。

### Q：如何设计「模型路由」？

**答：** 按 **任务类型/语言/成本/延迟** 路由到不同模型；小分类器或规则优先；记录 **路由决策** 便于优化；失败 **fallback**。

### Q：百万级文档检索如何扩展？

**答：** **分片/分区**（按租户）、**近实时索引管道**、**缓存热门 query**、向量库 **水平扩展**、**批嵌入**与 **离线重建**流程。

## 17. 行为面与项目深挖（技术向）

### Q：介绍你最熟悉的 Agent/RAG 项目。

**答（STAR 骨架）：**  
- **S**：业务场景与用户痛点。  
- **T**：你的职责与技术目标（延迟、准确率、成本）。  
- **A**：架构选型（模型、向量库、队列）、关键难点（分块、工具鉴权、流式）。  
- **R**：指标前后对比、线上事故与复盘。  
准备 **1 个成功细节 + 1 个失败与改进**，比罗列名词分高。

### Q：你和产品经理/算法怎么协作？

**答：** 明确 **验收指标**（准确率、满意度、成本）；**用例表**驱动开发；**bad case** 闭环；**版本化** prompt 与检索参数。

### Q：你如何学习新模型/新框架？

**答：** 读 **官方文档 + 最小复现**；关注 **changelog** 与 **breaking**；用小项目验证后再引入生产。

### Q：如何做 A/B 测试 Prompt 或模型？

**答：** **流量分桶**（用户级或会话级稳定哈希）；统一 **日志与指标**（任务成功率、满意度、成本）；**同一会话不混用**版本以免体验跳变；统计上注意 **样本量与置信区间**。

### Q：多租户 Agent 系统要注意什么？

**答：** **数据隔离**（向量库 namespace、DB 行级租户 id）、**配额与限流按租户**、**密钥与模型路由按租户**、**审计日志**可追溯到租户；避免**跨租户缓存键**错误。

### Q：数据驻留与跨境合规？

**答：** 部分行业要求数据**不出境**或**指定区域**；选**区域化部署**的模型 API 与向量库；合同与 **DPA**；日志脱敏与**留存周期**写入政策。

## 18. 增补：Python 进阶与踩坑

### Q：`__slots__` 有什么用？

**答：** 限制实例 `__dict__`，**省内存**、加快属性访问；缺点：不能再动态加属性、多重继承要小心。Agent 里大量小对象时偶用。

### Q：`dataclass` 与 `NamedTuple`？

**答：** `dataclass` 可变、默认生成 `__init__` 等，适合配置与中间结构；`NamedTuple` 不可变、元组式unpack。消息对象若需原地改字段用 dataclass。

### Q：`import` 慢怎么排查？

**答：** **重型库延迟导入**；用 **`python -X importtime`** 或 profiler；生产用 **多 worker 预加载** 分摊。

### Q：内存泄漏常见原因？

**答：** **全局缓存无限增长**、**循环引用 + 错误 `__del__`**、**未关闭连接**、**lru_cache 无界**。Agent 长会话注意 **messages 列表**与 **检索缓存 TTL**。

### Q：`pickle` 能用于 Agent 消息持久化吗？

**答：** **不安全**（反序列化 RCE），且跨版本脆弱。**JSON/MessagePack** + 明确 schema 更安全。

---

## 19. 增补：数据库与 ORM

### Q：N+1 查询是什么？怎么避免？

**答：** 循环里每条记录再查关联表，一发变 N 发。用 **join / selectinload / prefetch**，或 **批量查询**再内存组装。

### Q：索引什么时候失效？

**答：** **对列做函数**、**隐式类型转换**、**前导模糊 like '%x'**、**选择性过低**的小表全表扫更便宜。Agent 日志查询要按实际执行计划看。

### Q：乐观锁 vs 悲观锁？

**答：** **乐观**：带 version 字段更新时比对，冲突则重试，适合读多写少。**悲观**：`SELECT FOR UPDATE`，适合强冲突写路径。

## 20. 增补：向量与嵌入细节

### Q：Embedding 模型怎么选？

**答：** 看 **语言**（中英混合用多语模型）、**维度**（与库兼容）、**领域**（代码/医疗可选专用）、**延迟与成本**（自托管 vs API）。要 **同一索引内模型一致**，换模型需 **全量重嵌入**。

### Q：余弦相似度与欧氏距离？

**答：** 向量**已 L2 归一化**时余弦与欧氏可单调相关；未归一则语义上常用 **余弦** 或 **内积**（视实现）。面试说清「是否归一」。

### Q：HNSW / IVF 等索引直觉？

**答：** **HNSW**：图结构近似最近邻，**查询快、建索引重**。**IVF**：聚类分桶再搜，**省内存、参数敏感**。选型看数据规模与延迟 SLA，以**压测**为准。

## 21. 增补：框架与「不用框架」

### Q：LangChain 类框架的价值与代价？

**答：** **价值**：快速拼链、生态集成、原型快。**代价**：抽象厚、版本变动、黑盒难 debug。生产常 **核心路径手写**（HTTP+SDK），框架用于 **非关键路径** 或 **内部工具**。

### Q：什么时候直接调 OpenAI/兼容 HTTP？

**答：** 链路简单、要**完全控制消息与重试**、强依赖**稳定延迟**与**可观测字段**时，**SDK/httpx 直调**更清晰。

## 22. 增补：部署、容器与弹性

### Q：Agent API 容器化注意什么？

**答：** **镜像分层**减小体积；**非 root**；**健康检查**要测依赖（模型网关、向量库）还是只测进程；**graceful shutdown** 让进行中的流式请求收尾；**环境变量**注入密钥。

### Q：无状态服务 vs 有状态？

**答：** **API worker 无状态**，会话放 **Redis/DB**；**WebSocket** 需 **sticky** 或 **pub/sub 层**；**长任务**出站列队。

### Q：K8s 对 Agent 服务一般怎么用？

**答：** **Deployment + HPA**（CPU/自定义指标如队列长度）；**ConfigMap/Secret**；**探针** liveness/readiness；**资源限制**防 OOM；有状态组件（向量库）常用 **托管服务** 而非自管集群。

## 23. 增补：认证、授权与 API 设计

### Q：JWT 在 Agent BFF 里注意什么？

**答：** **短有效期 + refresh**；**aud/iss** 校验；权限 **scope**；**不在 payload 存敏感明文**；服务端 **吊销** 需黑名单或会话表。

### Q：API Key vs OAuth？

**答：** **Key** 适合服务间简单场景；**OAuth2/OIDC** 适合用户委托与细粒度 scope。Agent **调第三方工具**常要 **OAuth 刷新令牌**安全存储。

### Q：如何设计「工具权限模型」？

**答：** **RBAC/ABAC**：角色绑定允许的工具集合；**每次 tool 调用**在服务端再校验 **租户 + 资源 id**；禁止模型 **自选 endpoint URL**。

## 24. 增补：流式与 UX

### Q：流式输出中途失败怎么对用户交代？

**答：** **SSE/ chunk 里发 error 事件**或最终补一条 **完整错误**；前端 **可重试**与 **断点续传**（难，通常重试整轮）；服务端 **记录已输出 token 数**便于排障。

### Q：首 Token 延迟（TTFT）优化？

**答：** **减小首包前处理**（检索并行、缓存）；**更快模型路由**；**连接池复用**；**边缘部署**；**prompt 缩短**。

## 25. 增补：评测与数据飞轮

### Q：LLM-as-judge 有什么问题？

**答：** **位置偏见**、**风格偏见**、**与裁判模型同源**；要 **多裁判**、**盲测**、**人工抽检校准**。

### Q：如何做「坏例」收集与迭代？

**答：** 线上 **点赞/点踩**、**人工标注台**、统一 **bad case 库**；每周 **回归** 新 prompt/检索参数是否在黄金集上 **不退化**。

## 26. 增补：更多系统设计变体

### Q：设计「代码仓库问答」Agent。

**答：** **索引**：按文件/符号 chunk，保留路径与 commit；**检索**：混合关键词+向量；**工具**：读文件、搜符号、**只读 git**；**安全**：**沙箱执行**任何代码建议，默认不执行；**大仓库**要 **增量索引** webhook。

### Q：设计「语音客服」链路（简述）。

**答：** ASR → **意图/槽位**（可小模型）→ Agent（检索+工具）→ TTS；关注 **流式 ASR**、**barge-in**、**延迟预算**；**PII** 在语音与文本链路全程脱敏。

### Q：设计「批量文档标签/分类」流水线。

**答：** **批处理队列**、**固定 schema 结构化输出**、**失败分区重试**、**人工抽检队列**、**成本封顶**（每文档 max token）；**幂等**（文档版本号）。

### Q：「实时知识」与 RAG 矛盾怎么缓解？

**答：** **短 TTL 缓存** + **高频源 API 直查工具**（不走向量）；向量库 **增量更新**；对时效问题 **显式展示「知识截止时间」**。

## 28. 增补：OpenAI / 兼容 API 常见坑

### Q：`max_tokens` 与上下文窗口？

**答：** 输出长度受 **`max_tokens`** 与**剩余上下文**共同限制；长 system + 长 RAG 会导致**可用输出变短**甚至**请求被拒**。要在服务端做 **token 预算**。

### Q：`stop` 序列？

**答：** 生成遇到指定字符串即停；用于 **结构化分隔符** 时别与正文冲突。

### Q：429 / 5xx 怎么处理？

**答：** **429**：降速、排队、换池或升配额；读 **`retry-after`**。**5xx**：可重试但要有 **上限**；**流式**半包要定义 **客户端重组或放弃策略**。

## 29. 增补：多模态与 Agent（简述）

### Q：图像输入在架构上多什么环节？

**答：** **上传与存储**（对象存储）、**病毒/内容扫描**、**多模态模型**或 **OCR+文本模型** 两路径；**成本与延迟**通常高于纯文本；注意 **PII**（证件照等）。
